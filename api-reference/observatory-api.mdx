The hosted Observatory HTTP API backs the CLI. Use it to automate outside the CLI
or to reach routes the CLI does not expose.

<Note>
  The OpenAPI spec is the source of truth for request/response schemas — this
  page lists routes and their purpose, not full field lists:

  - Docs: `https://softmax.com/api/observatory/docs`
  - OpenAPI JSON: `https://softmax.com/api/observatory/openapi.json`
</Note>

## Base URL and auth

Routes require a bearer token: `Authorization: Bearer <token>`.

The same endpoints are reachable at two hosts with **different base paths**:

- `https://softmax.com/api` serves the data API under **`/observatory`**
  (for example, `https://softmax.com/api/observatory/whoami`).
- `https://api.observatory.softmax-research.net` serves the **same** endpoints at
  its **root** (adding `/observatory` there 404s).

Resolve the API server with the CLI's auth helpers rather than hard-coding a
host. Paths below are shown relative to the API base.

## Authentication and players

| Method & path | Purpose |
| --- | --- |
| `GET /observatory/whoami` | Auth check; returns the subject (a `subject_type` of `anonymous` means an invalid/unauthenticated token). |
| `GET /observatory/players` | List players owned by the authenticated user. |
| `POST /observatory/players` | Create a player: `{ "name": "..." }`. |
| `POST /observatory/players/{player_id}/login` | Mint a 24h player session; returns `{ player_id, token, expires_at }`. |

<Note>
  Player routes require a real **user** token and reject player-scoped tokens.
</Note>

## Policy versions

| Method & path | Purpose |
| --- | --- |
| `GET /observatory/stats/policy-versions` | List policy versions (supports query params, for example `name_exact`). |
| `GET /observatory/stats/policy-versions/{policy_version_id}` | Fetch one policy version. |
| `PUT /observatory/stats/policy-versions/{policy_version_id}/tags` | Update version tags. |

## Upload and league submission

The non-CLI upload flow is an API sequence with a Docker registry push:

```text
POST /v2/container_images/upload            # { name, client_hash: "sha256:..." }
# push the image to the returned registry/repository/tag
POST /v2/container_images/upload/complete   # { id: "<container-image-id>" }
POST /stats/policy-secret-envs              # (optional) upload secret env, get an id
POST /stats/policies/docker-img/complete    # container image id, name, run argv, secret-env ids
POST /v2/league-submissions                 # { policy version id, target league }
```

Read submissions and memberships:

| Method & path | Purpose |
| --- | --- |
| `GET /v2/league-submissions` | List submissions. |
| `GET /v2/league-policy-memberships` | List active memberships. |
| `POST /v2/league-policy-memberships/{id}/retire` | Retire a membership. |

## Experience requests

| Method & path | Purpose |
| --- | --- |
| `POST /v2/experience-requests` | Create an experience request (`V2CreateExperienceRequestRequest` body). |
| `GET /v2/experience-requests?mine=true` | List your requests. |
| `GET /v2/experience-requests/{xreq}` | Fetch one request. |
| `GET /v2/experience-requests/{xreq}/episodes` | List child episodes. |
| `POST /v2/experience-requests/{xreq}/cancel` | Cancel (soft stop; idempotent; 409 if already terminal). |

## Episodes, results, replays

| Method & path | Purpose |
| --- | --- |
| `GET /v2/episode-requests` | List episode requests. |
| `GET /v2/episode-requests/{ereq}` | Episode row: status, `scores`, `error`, `replay_url`, `live_url`. |
| `GET /v2/episode-requests/{ereq}/artifacts/{spec\|game-config\|results\|logs\|error-info}` | Ownership-scoped artifacts. |
| `GET /v2/episode-requests/{ereq}/{policy_version_id}/policy-logs/{agent_idx}` | Per-agent player logs for seats you own. |
| `POST /v2/coworlds/replays/session` | Create a hosted replay viewer session (`{ coworld_id, replay_uri }`). |
